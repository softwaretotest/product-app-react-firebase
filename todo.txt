Draft autosave (เพิ่มอีกนิด แต่ impact สูงมาก) กัน Form Data disappear during editing

เก็บฟอร์มไว้ก่อน reload:

// save
localStorage.setItem("productFormDraft", JSON.stringify(form));

// load ตอน mount
const draft = localStorage.getItem("productFormDraft");
if (draft) setForm(JSON.parse(draft));

Idea for longterm scale 
เราจะมีสามแบบ 
1. Generic UI dump / reusable 
2. Generic UI smart 
3. Generic UI Default (มีส่วน UI dump/reusable + smart) 
สําหรับ Entity ที่มาแบบซ้ําซ้ํา product order customer CRUD 

4.แต่สําหรับ Entity ที่พิเศษอย่างเช่น UI Special e.g. Bill, Email etc. 
ที่ มี specific field เหนือจาก UI default เราจะใช้ฟังชัน prototype 
อย่าง JavaScript หรือสิ่งที่คล้ายๆ object herachy กันมา inherit default 
แล้วก็เพิ่มเข้าไปใน UI special . 
ฟังดูตอนนี้เหมือน over engineering 
แต่เราคุยถึงอนาคตที่มันยังไกลและมาไม่ถึงครับ 
แต่นี่คือทางที่เราจะ scale โพรดักต์แล้วไม่ให้ 
codes มหาศาลเกิดขึ้นได้ซ้ําซ้อน
